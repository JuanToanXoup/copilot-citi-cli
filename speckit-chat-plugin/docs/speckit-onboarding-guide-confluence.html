What is Speckit Chat?

Speckit Chat is an IntelliJ plugin that brings the Spec-Kit pipeline directly into your IDE. It connects to GitHub Copilot Chat in Agent mode, letting you run specialized AI agents that take you from a one-line feature idea all the way to implemented, tested code and GitHub issues.

1.Constitution→ 2.Specify→ 3.Clarify→ 4.Plan→ 5.Tasks→ 6.Checklist→ 7.Analyze→ 8.Implement→ 9.Issues

Each step produces artifacts that feed into the next. Auto-handoffs keep the flow moving.

Install the Plugin

  1. Download the plugin ZIP — Download Speckit-Chat-0.1.0.zip from the link at the top of this page or from the attachment below.
  2. Open Plugin Settings — IntelliJ IDEA menu → Settings... (Cmd+, on macOS, Ctrl+Alt+S on Windows/Linux).
  3. Install from Disk — Go to Plugins, click the gear icon (&9881;) → Install Plugin from Disk...
  4. Select the ZIP file — Navigate to the downloaded Speckit-Chat-0.1.0.zip and click Open.
  5. Restart IntelliJ — Click Restart IDE. The Speckit Chat tool window appears in the bottom panel.

Usage

Running Your First Agent

  1. Open the Sessions tab — The agent dropdown lists all available agents from .github/agents/ plus bundled defaults.
  2. Select an agent — Start with constitution to establish project principles, then specify to create your first feature spec.
  3. Enter arguments and send — Type your input and click send (or Ctrl+Enter). The prompt is sent to Copilot Chat in Agent mode.
  4. Track progress — The run table shows status, duration, branch, and session ID. Double-click a row to switch to that Copilot Chat session.

Typical Workflow

  1. Discovery — Establish your constitution — Go to the Discovery tab, answer the project property questions, and generate your constitution.md.
  2. Pipeline — Select a feature — Open the Pipeline tab, select a feature directory, and validate each artifact step by step.
  3. Pipeline — Run each step — Click Run Step for each pipeline stage. Arguments are autofilled with sensible defaults and persisted per feature.
  4. Sessions — Monitor and interact — Switch to Sessions to see all running and completed agents. Double-click any row to open that Copilot Chat session.
Agent Mode Required

Copilot Chat must be in Agent mode (not Ask or Edit mode) for Speckit agents to function. The plugin enables this automatically when sending prompts, but if you interact with Copilot Chat directly, ensure Agent mode is selected.

Agent Details

constitution — Define governance principles

The constitution agent establishes the engineering principles, constraints, and standards that all subsequent agents must follow. Run this first, ideally via the Discovery tab's guided Q&A.

  • Input: Project context and user-supplied engineering principles
  • Output: .specify/memory/constitution.md
  • Default arguments: Refer to the ./specify/memory/discovery.md for project properties
 specify — Transform a feature description into a spec

Takes a natural-language feature description and produces a structured specification with requirements, acceptance criteria, and a git feature branch.

  • Input: Feature description text
  • Output: spec.md, checklists/requirements.md, new git branch
  • Auto-handoff: If the spec contains unresolved markers, automatically chains to the Clarify agent
  • Default arguments: 100% Unit Test Case coverage for all microservices code.
 clarify — Resolve underspecified areas

Scans the spec for ambiguous or incomplete areas and generates targeted clarification questions. Updates the spec with resolutions.

  • Input: spec.md with unresolved markers
  • Output: Updated spec.md with clarifications resolved
 plan — Generate technical design

Produces a comprehensive technical plan with architecture decisions, data models, API contracts, and research notes.

  • Input: spec.md, constitution.md
  • Output: plan.md, research.md, data-model.md, contracts/
  • Auto-handoff: Chains to Tasks + Checklist agents
 tasks — Break plan into phased task list

Decomposes the plan into dependency-ordered, phased implementation tasks with clear acceptance criteria.

  • Input: plan.md, spec.md
  • Output: tasks.md
  • Auto-handoff: Chains to Analyze + Implement agents
  • Default arguments: all tasks
 checklist — Generate requirement checklists

Creates domain-specific checklists that gate implementation quality. Each checklist item maps back to a spec requirement.

  • Input: spec.md
  • Output: checklists/[domain].md files
 analyze — Cross-artifact consistency check

Read-only analysis that validates consistency across all artifacts (spec, plan, tasks, constitution). Reports gaps and contradictions without modifying files.

  • Input: tasks.md, spec.md, plan.md, constitution.md
  • Output: Consistency report (stdout only, no file changes)
 implement — Execute tasks with TDD

Works through the task list phase-by-phase, writing code with test-driven development and checking off checklist items as it goes.

  • Input: tasks.md, plan.md, checklists
  • Output: Source code, updated tasks.md with completed items
  • Default arguments: all remaining tasks
 taskstoissues — Convert tasks to GitHub issues

Reads tasks.md and creates one GitHub issue per task, preserving dependencies and labels.

  • Input: tasks.md, GitHub remote configured
  • Output: GitHub issues (one per task)
  • Default arguments: all tasks

Requires a GitHub remote with push access. Ensure your repository has a configured remote before running this agent.

Customization

50%

Override Bundled Agents

Place custom .agent.md files in .github/agents/. Project-level files take priority over bundled defaults.

Agent file naming convention: speckit.<name>.agent.md

 50%

Refresh Agents

Click the … menu in the Sessions tab and select Refresh Agents to reload without restarting the IDE.

You can also re-run Init Speckit to fetch the latest agent definitions from the Spec-Kit release.

Pipeline Argument Persistence

The Pipeline tab remembers arguments you've entered per feature and per step. Arguments are persisted across IDE restarts using IntelliJ's project-level properties. Click Reset next to the arguments field to restore the default autofill value.
